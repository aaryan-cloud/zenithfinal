      07 · ROUND 1 BLUEPRINT

      How We Build the Full Celestial Eye

      This demo runs on a hand-built ephemeris engine. Here is the production flight plan.

      A

      Data Strategy

      • NASA JPL Horizons — authoritative planetary & small-body ephemerides via REST.
      • Open Notify / WhereTheISS — live ISS position & crew telemetry (already wired into this demo).
      • CelesTrak TLEs + SGP4 — 9,000+ tracked satellites propagated client-side.
      • Open-Meteo Geocoding — keyless global city search (live in this demo).
      B

      Tech Stack

      • satellite.js (SGP4) — already live in this build, propagating real TLE element sets.
      • astronomy-engine — VSOP87-class Sun/Moon/planet ephemeris, also live in this build.
      • Next.js + React + TypeScript + Tailwind — production migration path, typed and edge-cached.
      • CesiumJS photoreal globe; Leaflet fallback for low-power devices.
      C

      Feasibility

      • Week 1 Data layer: Horizons + CelesTrak ingestion, pass-prediction service.
      • Week 2 Cesium globe, radar dashboard, location pipeline.
      • Week 3 AI Guide (LLM over structured ephemeris), Time Travel, events engine.
      • Week 4 Polish, accessibility, load testing, deploy on Vercel edge.
      D

      Innovation

      • AI Cosmic Guide — ephemeris-grounded narration; zero hallucinated skies.
      • Time Travel Mode — one scrubber rewinds or fast-forwards the entire universe.
      • Near-zenith detection — the Eye always knows what is most directly overhead.
      • AR Sky View & pass push-alerts (future) — point your phone at the sky, get “ISS in 10 min” pings.
      E

      Accuracy Approach

      • SGP4 propagation of NORAD two-line elements — the same model used by ground stations.
      • VSOP87-class planetary theory via astronomy-engine: arc-second-level geometry, refraction-corrected horizons.
      • Topocentric corrections — every altitude/azimuth is computed for the exact observer coordinates.
      • Validation — cross-checked against Heavens-Above pass times and the Polaris-altitude ≈ latitude invariant.
      F

      Offline Fallback Strategy

      • Cached TLE set ships in the bundle — SGP4 keeps working with zero connectivity.
      • Dual ephemeris engines — if the vendored library fails, a hand-built Keplerian engine takes over seamlessly.
      • Offline city index — 60 world cities searchable without a network.
      • Live upgrades, never dependencies — CelesTrak, wheretheiss.at and geocoding enhance the data when reachable.
